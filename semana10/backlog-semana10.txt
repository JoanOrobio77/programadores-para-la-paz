  ✓ Logout limpia sesión

[ALTA] TASK 7.1: Crear modelo de incidencias
Descripción: Schema de tabla incidents: id, titulo, descripcion, ubicacion, foto, estado, ciudadanoId, createdAt
Historia de usuario: Como ciudadano quiero registrar una incidencia para dejar evidencia del caso.
Criterios de aceptación:
  ✓ Ciudadano puede crear incidencia sin ser experto
  ✓ Permite adjuntar foto (geolocalización opcional)
  ✓ Clasificación de tipo (seguridad, servicios, infraestructura, etc.)
  ✓ Validación de datos sensibles antes de guardar
Estimación: 5 puntos

[ALTA] TASK 7.2: API para reportar incidencia
Descripción: POST /api/incidents con validación y almacenamiento seguro
Historia de usuario: Como ciudadano quiero registrar una incidencia para dejar evidencia del caso.
Criterios de aceptación:
  ✓ Endpoint POST /api/incidents
  ✓ Validación de datos no sensibles
  ✓ Asignación automática de ID único
  ✓ Retorna confirmación con número de reporte
Estimación: 5 puntos

[MEDIA] TASK 7.3: Notificación en Telegram para coordinadores
Descripción: Alertas a grupo de Telegram cuando hay nueva incidencia
Historia de usuario: Como coordinador quiero recibir una alerta en Telegram para revisar un nuevo reporte.
Criterios de aceptación:
  ✓ Envío automático al registrar incidencia
  ✓ Mensaje con: ID reporte, tipo, ubicación, estado
  ✓ Botón para revisar en plataforma
  ✓ Solo información pública, sin datos sensibles
  ✓ Respetar horario de notificaciones
Estimación: 5 puntos

[MEDIA] TASK 7.4: Panel de reportes para seguimiento
Descripción: Consulta de reportes con filtros de estado y tipo
Historia de usuario: Como miembro del equipo quiero consultar reportes para hacer seguimiento.
Criterios de aceptación:
  ✓ GET /api/incidents con filtrado por estado
  ✓ Filtro por tipo de incidencia
  ✓ Filtro por fecha
  ✓ Paginación de resultados
  ✓ Estadísticas: pendientes, en proceso, resueltas
  ✓ Solo mostrar datos no sensibles
Estimación: 5 puntos

[MEDIA] TASK 7.5: Actualización de estado de incidencia
Descripción: Permitir cambiar estado (pendiente → en proceso → resuelto)
Historia de usuario: Como coordinador quiero actualizar el estado de una incidencia para informar al ciudadano.
Criterios de aceptación:
  ✓ PATCH /api/incidents/:id/status
  ✓ Solo coordinador/admin puede actualizar
  ✓ Historial de cambios
  ✓ Notificación al ciudadano reportante
  ✓ Permite agregar comentario de seguimiento
Estimación: 5 puntos
